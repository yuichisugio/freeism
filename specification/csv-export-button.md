# CSVエクスポートボタン仕様書

## 1. 概要

CSVエクスポートボタンは、グループ管理者やアプリケーション所有者がグループ内のタスクデータや分析結果データをCSV形式でエクスポートするための機能を提供します。これにより、ユーザーはデータを外部ツールで分析したり、バックアップとして保存したりすることができます。

### 1.1 用途

- タスク情報の一括エクスポート
- 分析結果データの一括エクスポート
- エクスポートしたデータの外部ツールでの分析
- データのバックアップ

### 1.2 目的

- データの可視化と分析の促進
- 外部ツールとの連携の簡易化
- データのバックアップと保存
- レポート作成の効率化

## 2. アクセス権限

- **グループオーナー**：

  - 自身が所有するグループのデータのみエクスポート可能
  - グループ内のすべてのタスクと分析結果にアクセス可能

- **アプリケーションオーナー**：

  - 全ての機能を利用可能
  - システム全体のあらゆるグループのデータにアクセス可能
  - 制限なしですべての操作が許可される

- **一般ユーザー**：
  - アクセス不可
  - エクスポート機能へのアクセス自体が制限される
  - 権限不足時には適切なエラーメッセージを表示

### 2.1 権限チェックロジック

1. セッションからユーザーIDを取得
2. アプリケーションオーナー権限チェック
   - ユーザーの `isAppOwner` フラグを検証
3. グループオーナー権限チェック
   - グループメンバーシップテーブルで `role === 'OWNER'` を検証
4. いずれかの権限が確認できた場合のみ機能を有効化

## 3. エクスポートタイプ

エクスポート機能では以下の2種類のデータタイプが利用可能です：

### 3.1 タスクデータ（TASK）

**説明**: グループ内のタスク情報をCSV形式でエクスポートします。

**エクスポート目的**:

- **分析用(ANALYSIS)**: 完了済みタスク（TASK_COMPLETED）のみをエクスポート
- **閲覧用(VIEWING)**: すべてのステータスのタスクをエクスポート

**エクスポートされるフィールド**:

- `タスクID`: タスクの一意識別子
- `タスク内容`: タスクの詳細説明
- `参照`: 参考URLなどの外部リンク
- `証拠情報`: タスク完了の証拠となる情報
- `ステータス`: タスクの現在のステータス
- `貢献ポイント`: 固定された貢献ポイント値
- `評価者`: 固定評価者の名前
- `貢献タイプ`: "REWARD"または"NON_REWARD"
- `作成者`: タスク作成者の名前
- `報告者`: タスク報告者の名前（複数の場合はカンマ区切り）
- `実行者`: タスク実行者の名前（複数の場合はカンマ区切り）
- `作成日`: タスクの作成日（YYYY-MM-DD形式）
- `更新日`: タスクの最終更新日（YYYY-MM-DD形式）

**処理フロー**:

1. 期間（開始日・終了日）の指定
2. エクスポート目的（分析用/閲覧用）の選択
3. サーバーへのデータリクエスト
4. データのCSV変換
5. CSVファイルのダウンロード

**制限事項**:

- 最大エクスポート期間は6ヶ月まで
- 未来の日付は選択不可
- 開始日は終了日より前でなければならない

### 3.2 分析結果データ（ANALYTICS）

**説明**: グループ内の貢献度評価結果をCSV形式でエクスポートします。

**フィルターオプション**:

- **FIX済みのみ**: ステータスが「POINTS_AWARDED」のタスクのみをエクスポート

**エクスポートされるフィールド**:

- `分析ID`: 分析の一意識別子
- `タスクID`: 関連するタスクの一意識別子
- `貢献ポイント`: 付与された貢献ポイント
- `評価ロジック`: 評価の理由や根拠
- `評価者ID`: 評価者の一意識別子
- `評価者名`: 評価者の名前
- `タスク内容`: 評価対象タスクの詳細
- `参照情報`: 参考URLなどの外部リンク
- `証拠情報`: タスク完了の証拠情報
- `ステータス`: タスクの現在のステータス
- `貢献タイプ`: "REWARD"または"NON_REWARD"
- `タスク報告者`: タスク報告者の名前
- `タスク実行者`: タスク実行者の名前
- `タスク作成者`: タスクを作成したユーザーの名前
- `グループ目標`: 関連するグループの目標
- `評価方法`: グループで採用している評価方法
- `作成日`: 評価が行われた日付（YYYY-MM-DD形式）
- `評価日`: FIX済みの場合のみ、評価が確定した日（YYYY-MM-DD形式）

**処理フロー**:

1. ページ番号の指定（1ページあたり200件）
2. FIX済みのみのオプション選択
3. サーバーへのデータリクエスト
4. 評価者ごとのデータグループ化
5. 評価者が1人の場合はCSV、複数の場合はZIPファイルでダウンロード

**制限事項**:

- 1ページあたり200件のデータ制限
- 最大5ページまでのページング
- 大量データの場合は複数回のエクスポートが必要

## 4. ユーザーインターフェース

### 4.1 エクスポートボタン

1. **ボタンの表示位置**

   - グループページのタスク一覧上部
   - 右上のアクションエリアに配置
   - 他のアクションボタン（追加、フィルターなど）と並べて表示

2. **ボタン外観**

   - アイコン: ダウンロードアイコン
   - ラベル: 「CSVエクスポート」
   - 色: アプリのプライマリカラー（青系）
   - サイズ: 標準ボタンサイズ
   - ホバー効果: わずかに明るくなる
   - プレス効果: わずかに暗くなる

3. **表示条件**

   - グループオーナーまたはアプリケーションオーナーの場合のみ表示
   - 権限がない場合は表示しない（非表示、無効化ではなく完全に非表示）

4. **アクセシビリティ**
   - アイコンとテキストの組み合わせで識別しやすさを向上
   - aria-label属性を使用して支援技術に対応
   - 色だけでなく形状でも区別できるデザイン
   - キーボードでのフォーカスと操作が可能（tabキーでのナビゲーション）
   - ハイコントラストモードでも視認性を確保

### 4.2 ボタン動作

1. **クリック時の挙動**

   - エクスポートモーダルを開く
   - モーダル外をクリックまたはESCキーでモーダルを閉じる
   - モーダル表示中は背景をオーバーレイで薄暗くする

2. **アニメーション効果**

   - クリック時: 軽いバウンスエフェクト
   - モーダル開閉: スムーズなフェードイン/アウト
   - 処理中: スピナーアイコンの表示

3. **エラー時の挙動**
   - アクセス権限エラー: トースト通知で権限がない旨を表示
   - サーバーエラー: トースト通知でエラーメッセージを表示
   - 回復可能なエラー: モーダル内でのエラー表示とリトライオプション

## 5. 機能要件

### 5.1 エクスポートモーダル

- **モーダルサイズ**:

  - 幅: 最大600px
  - 高さ: コンテンツに合わせて可変（最大95vh）
  - レスポンシブデザイン: モバイル画面では全画面表示

- **モーダル構成**:

  - ヘッダー: タイトル「データをエクスポート」とグループ名
  - ステッププログレス: 現在のステップを視覚的に表示
  - コンテンツエリア: ステップごとの設定UI
  - フッター: 「前へ」「次へ」「エクスポート」などのアクションボタン

- **ステップ構成**:
  - ステップ1: データタイプ選択（タスクまたは分析結果）
  - ステップ2: タスクの場合は期間指定、分析結果の場合はページ指定

### 5.2 データ型選択ステップ

- **タスクデータカード**:

  - アイコン: ファイルアイコン
  - タイトル: 「タスクデータ」
  - 説明: タスクデータのエクスポート内容説明
  - 選択時の視覚的フィードバック: 青色の枠とハイライト効果

- **分析結果データカード**:

  - アイコン: チャートアイコン
  - タイトル: 「分析結果データ」
  - 説明: 分析結果データのエクスポート内容説明
  - 選択時の視覚的フィードバック: 紫色の枠とハイライト効果

- **エクスポート目的選択**（タスクデータの場合）:

  - 分析用: 完了済みタスクのみ
  - 閲覧用: 全ステータスのタスク

- **フィルターオプション**（分析結果データの場合）:
  - FIX済みのみ: チェックボックスで選択可能

### 5.3 期間指定ステップ（タスクデータ）

- **日付選択UI**:

  - 開始日選択: カレンダーピッカー
  - 終了日選択: カレンダーピッカー
  - 日付表示形式: YYYY年MM月DD日
  - 無効な日付: グレーアウト表示

- **日付制約**:

  - 最大期間: 6ヶ月
  - 未来日付: 選択不可
  - 開始日 > 終了日: エラーメッセージ表示

- **期間サマリー**:
  - 選択した期間の表示: 「選択された期間: XXXX年XX月XX日 〜 XXXX年XX月XX日」
  - 期間の妥当性チェック: エラーがある場合は続行不可

### 5.4 ページ指定ステップ（分析結果データ）

- **ページ選択UI**:

  - ドロップダウンで1〜5ページから選択
  - 各ページの範囲表示: 「Nページ目 (X〜Y件)」
  - デフォルト: 1ページ目

- **ページサマリー**:
  - 選択したページの表示: 「選択されたページ: N ページ目 (X〜Y件)」
  - フィルター状態表示: 「FIX済みの分析結果のみ」（有効時）

### 5.5 エクスポート処理

- **処理表示**:

  - エクスポートボタンのスピナー表示
  - 「エクスポート中...」テキスト表示
  - エクスポート中はボタン無効化

- **完了通知**:

  - 成功時: トースト通知「データを正常にエクスポートしました」
  - エラー時: トースト通知でエラーメッセージ表示

- **ファイル形式**:

  - タスクデータ: 単一CSVファイル
  - 分析結果データ（単一評価者）: 単一CSVファイル
  - 分析結果データ（複数評価者）: ZIPファイル（評価者ごとのCSV）

- **ファイル命名規則**:
  - タスクデータ: `[グループ名]_tasks_[all/analysis]_[YYYYMMDD].csv`
  - 分析結果データ（単一）: `[グループ名]_analytics_[YYYYMMDD]_[評価者名].csv`
  - 分析結果データ（複数）: `[グループ名]_analytics_[YYYYMMDD].zip`

## 6. 技術仕様

### 6.1 フロントエンド実装

- **ボタンコンポーネント**:

  - ShadcnUI Button コンポーネントを使用
  - アイコンはlucide-reactのDownloadアイコンを使用
  - スタイルは Tailwind CSS で定義

- **モーダルコンポーネント**:

  - ShadcnUI Dialog コンポーネントを使用
  - アニメーションはframer-motionライブラリを使用
  - ステップ遷移時の方向によるアニメーション分岐

- **日付選択**:

  - ShadcnUI Calendar, Popoverコンポーネントを使用
  - date-fns ライブラリで日付操作
  - 日本語ロケールを使用

- **CSV生成**:

  - PapaParse ライブラリでJSONからCSV変換
  - BlobとFileReaderAPIを使用してダウンロード
  - UTF-8エンコーディングでBOM無しCSV生成

- **ZIP生成**:
  - fflate ライブラリでZIPファイル生成
  - 複数CSVファイルを含むZIPアーカイブ作成
  - ファイル名は評価者名を使用

### 6.2 バックエンド実装

- **Server Actions**:

  - `exportGroupTask`: タスクデータをエクスポート
  - `exportGroupAnalytics`: 分析結果データをエクスポート

- **データ取得**:

  - Prisma クエリによるデータベースからの検索
  - 関連エンティティの一括取得と結合
  - データの整形と構造化

- **権限管理**:

  - NextAuth セッションを使用したユーザー認証
  - Prisma クエリによるグループ所有権の確認
  - エラー時は適切なエラーメッセージを返す

- **エラーハンドリング**:
  - try/catch による包括的なエラーハンドリング
  - 詳細なエラーログをコンソールに出力
  - クライアントに適切なエラーメッセージを返す

## 7. セキュリティ

- **アクセス制御**:

  - サーバーサイドでの権限検証
  - クライアントとサーバー両方で権限チェック
  - 認証されていないリクエストの拒否

- **データ保護**:

  - センシティブデータのフィルタリング
  - データ量の制限（ページングによる）
  - ダウンロードファイルへのアクセス制限

- **エラー処理**:
  - 詳細なエラーメッセージはログにのみ記録
  - クライアントには一般的なエラーメッセージを表示
  - スタックトレースの非公開

## 8. UIコンポーネント詳細

### 8.1 エクスポートボタン

```tsx
<Button
  variant="default"
  size="sm"
  className="gap-2 bg-blue-600 hover:bg-blue-700"
  onClick={handleOpenExportModal}
  aria-label="CSVデータのエクスポート"
>
  <DownloadIcon className="h-4 w-4" />
  CSVエクスポート
</Button>
```

### 8.2 ステッププログレス

```tsx
<div className="mt-5 mb-1 flex items-center">
  <div className="flex w-full items-center space-x-3">
    <StepCircle step={1} currentStep={state.step} />
    <StepLine isActive={state.step > 1} />
    <StepCircle step={2} currentStep={state.step} />
  </div>
</div>
```

### 8.3 アクションボタン

```tsx
<div className="flex items-center justify-between">
  <Button variant="outline" type="button" onClick={prevStep} disabled={state.step === 1} className="flex items-center gap-2">
    <ArrowLeft className="h-4 w-4" />
    前へ戻る
  </Button>

  {state.step === 2 ? (
    <Button type="button" onClick={handleExport} disabled={isExporting} className="flex items-center gap-2 bg-green-600 hover:bg-green-700">
      {isExporting ? (
        <>
          <Loader2Icon className="h-4 w-4 animate-spin" />
          エクスポート中...
        </>
      ) : (
        <>
          <DownloadIcon className="h-4 w-4" />
          エクスポート
        </>
      )}
    </Button>
  ) : (
    <Button type="button" onClick={nextStep} className="flex items-center gap-2">
      次へ進む
      <ArrowRight className="h-4 w-4" />
    </Button>
  )}
</div>
```

## 9. 将来の拡張性

- **追加のエクスポート形式**:

  - Excel (XLSX) 形式のサポート
  - JSON形式のサポート
  - PDF形式のレポート生成

- **高度なフィルタリング**:

  - タスクステータスによるフィルタリング
  - 特定ユーザーのタスクのみをエクスポート
  - カスタムフィールド選択によるエクスポート

- **自動エクスポート**:

  - 定期的な自動エクスポートのスケジュール
  - 特定のイベント（月末や評価完了時）に自動エクスポート
  - メールでの自動配信

- **データの可視化**:
  - エクスポート前のデータプレビュー
  - 簡易グラフやチャートの表示
  - インタラクティブなデータ探索
