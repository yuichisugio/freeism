# CSVアップロードモーダル仕様書

## 1. 概要

CSVアップロードモーダルは、グループ管理者やアプリケーション所有者がCSVファイルを通じてタスクや貢献度評価などのデータをシステムに一括登録するための機能を提供します。複数のデータタイプを処理でき、バルク操作によりワークフローの効率化を実現します。

### 1.1 用途

- タスク情報の一括登録
- 貢献度評価の一括実施
- 貢献度タイプの一括更新
- タスクステータスの一括更新

### 1.2 目的

- 手動データ入力の削減
- 大量データの効率的な処理
- ユーザーワークフローの簡素化
- データ整合性の確保

## 2. アクセス権限

- **グループオーナー**：

  - グループ関連の全てのCSVアップロード機能を利用可能
  - 自身が所有するグループのデータのみアクセス可能
  - グループ内のすべてのタスクとユーザーの操作が可能

- **アプリケーションオーナー**：

  - 全ての機能を利用可能
  - システム全体のあらゆるグループのデータにアクセス可能
  - 制限なしですべての操作が許可される

- **一般ユーザー**：
  - アクセス不可
  - モーダルへのアクセス自体が制限される
  - 権限不足時には適切なエラーメッセージを表示

### 2.1 権限チェックロジック

1. セッションからユーザーIDを取得
2. アプリケーションオーナー権限チェック
   - `checkAppOwner(userId)` 関数で確認
   - ユーザーの `isAppOwner` フラグを検証
3. グループオーナー権限チェック
   - `checkGroupOwner(userId, groupId)` 関数で確認
   - グループメンバーシップテーブルで `role === 'OWNER'` を検証
4. いずれかの権限が確認できた場合のみ機能を有効化

## 3. アップロードタイプ

モーダルでは以下の4種類のCSVアップロードタイプが利用可能です：

### 3.1 タスク報告（TASK_REPORT）

**説明**: ユーザーが行ったタスクを一括登録します。

**必須フィールド**:

- `task`: タスク内容（文字列、最大1000文字）
- `userName`: ユーザー名（システム内に存在するユーザー名）
- `date`: 実施日（YYYY-MM-DD形式）

**オプションフィールド**:

- `contributionType`: 貢献度タイプ（contributionType.REWARD または contributionType.NON_REWARD、デフォルト: contributionType.NON_REWARD）
- `reference`: 参考URL（有効なURL形式）
- `info`: 追加情報（文字列、最大500文字）

**処理フロー**:

1. CSVファイルのパース
2. 各行ごとにユーザー名からユーザーIDの解決
   - 存在しないユーザー名はエラーとして記録
3. 日付のフォーマット検証と変換
4. Prismaトランザクションによるタスクの一括作成
   - タスク、タスク報告者、実行者の関連レコードを作成
5. 成功/失敗結果の集計

**バリデーションルール**:

- ユーザー名：システム内に存在するユーザー名であること
- 日付：YYYY-MM-DD形式、過去の日付であること（未来日は不可）
- タスク内容：必須、1〜1000文字以内
- 貢献度タイプ：指定時は contributionType.REWARD または contributionType.NON_REWARD のみ許可
- 参考URL：指定時は有効なURL形式であること

### 3.2 貢献度評価（CONTRIBUTION_EVALUATION）

**説明**: 他のユーザーの貢献度を一括評価します。

**必須フィールド**:

- `taskId`: 評価対象のタスクID（システム内に存在するタスクID）
- `evaluatorName`: 評価者のユーザー名（システム内に存在するユーザー名）
- `evaluationLogic`: 評価ロジック（文字列、最大500文字）

**オプションフィールド**:

- `date`: 評価日（YYYY-MM-DD形式、デフォルト: 現在の日付）

**処理フロー**:

1. CSVファイルのパース
2. 各行ごとに以下の検証:
   - タスクIDの存在確認
   - 評価者のユーザー名からユーザーIDの解決
   - タスクとグループの関連性確認
3. 評価データのデータベース登録
   - 既存の評価がある場合は上書き
   - 新規評価の場合は作成
4. 成功/失敗結果の集計

**バリデーションルール**:

- タスクID：システム内に存在し、かつ該当グループに関連付けられていること
- 評価者：システム内に存在するユーザーであること
- 評価ロジック：必須、1〜500文字以内
- 日付：YYYY-MM-DD形式、過去または現在の日付であること

### 3.3 貢献度固定値（FIXED_CONTRIBUTION）

**説明**: タスクの貢献度を固定値で一括設定します。

**必須フィールド**:

- `taskId`: タスクID（システム内に存在するタスクID）
- `contributionType`: 貢献度タイプ（contributionType.REWARD または contributionType.NON_REWARD）

**オプションフィールド**:

- なし

**処理フロー**:

1. CSVファイルのパース
2. 各行ごとに以下の検証:
   - タスクIDの存在確認
   - タスクとグループの関連性確認
   - 貢献度タイプの検証
3. Prismaトランザクションによるタスク貢献度タイプの一括更新
4. 成功/失敗結果の集計

**バリデーションルール**:

- タスクID：システム内に存在し、かつ該当グループに関連付けられていること
- 貢献度タイプ：contributionType.REWARD または contributionType.NON_REWARD のみ許可
- タスクID重複：同一タスクIDが複数行に存在した場合、最後の行の値が適用される

### 3.4 タスクステータス（TASK_STATUS）

**説明**: タスクのステータスを一括更新します。

**必須フィールド**:

- `taskId`: タスクID（システム内に存在するタスクID）
- `status`: 新しいステータス（"PENDING"/"IN_PROGRESS"/"COMPLETED"/"CANCELED"）

**オプションフィールド**:

- なし

**処理フロー**:

1. CSVファイルのパース
2. 各行ごとに以下の検証:
   - タスクIDの存在確認
   - ステータス値の検証
   - タスクとグループの関連性確認
3. Prismaトランザクションによるタスクステータスの一括更新
4. 成功/失敗結果の集計

**バリデーションルール**:

- タスクID：システム内に存在し、かつ該当グループに関連付けられていること
- ステータス：TaskStatusの列挙型（"PENDING"/"IN_PROGRESS"/"COMPLETED"/"CANCELED"）の中から選択
- タスクID重複：同一タスクIDが複数行に存在した場合、最後の行の値が適用される
- ステータス遷移：任意のステータスから任意のステータスへの変更が許可される

## 4. ユーザーインターフェース

### 4.1 モーダル構成

1. **モーダルヘッダー**

   - タイトル: 「CSVアップロード」
   - 閉じるボタン: 右上にXアイコン
   - 幅: 最大800px
   - 高さ: コンテンツに応じて可変（最大80vh）

2. **アップロードタイプ選択セクション**

   - 各アップロードタイプのラジオボタン
     - 選択状態を視覚的に明確に表示
     - 権限がない場合は無効（グレーアウト）表示
   - タイプごとの説明文
     - 簡潔な説明とユースケース
     - アイコンによる視覚的な区別

3. **ファイルアップロードエリア**

   - ドラッグ＆ドロップゾーン
     - 境界線付きのエリア（点線）
     - ドラッグオーバー時のハイライト効果
   - ファイル選択ボタン: 「ファイルを選択」
   - フォーマット情報表示（詳細は4.2を参照）
   - サンプルCSVデータ表示: テーブル形式

4. **アップロードされたファイルリスト**

   - ファイル名: 左揃え
   - サイズ表示: 人間が読みやすい形式（例：2.3MB）
   - 削除ボタン: 各ファイル行の右端
   - 全削除ボタン: リスト下部（ファイルが複数ある場合のみ表示）
   - スクロール可能: 多数のファイルがある場合

5. **アクションボタン**
   - キャンセルボタン: 左側、グレー
   - アップロードボタン: 右側、プライマリカラー
     - アップロード中は無効化
     - ファイルが選択されていない場合は無効化

### 4.2 フォーマット情報表示

各アップロードタイプごとに以下の情報を表示：

- **必須フィールドの一覧と説明**

  - 赤色のアスタリスク（\*）で必須項目を明示
  - フィールド名と簡潔な説明
  - データ型や制約の説明

- **オプションフィールドの一覧と説明**

  - フィールド名と簡潔な説明
  - デフォルト値がある場合はその表示

- **サンプルCSVデータのテーブル**

  - ヘッダー行: 太字で表示
  - サンプルデータ行: 1行のみ表示
  - スクロール可能: 列が多い場合
  - カラム説明にはツールチップを表示

- **注意事項**（存在する場合）
  - 警告アイコン付きの枠内に表示
  - 特に注意すべき制約や条件を強調

## 5. 機能要件

### 5.1 ファイルアップロード

- **対応ファイル形式**: CSV形式のみ

  - MIME Type: `text/csv`
  - ファイル拡張子: `.csv`
  - 文字コード: UTF-8

- **最大ファイルサイズ**: 5MB

  - 超過時は即時エラー表示
  - アップロード前に検証

- **ファイル検証**:

  - アップロード前のヘッダー検証
    - 必須フィールドの存在確認
    - 大文字小文字を区別せずフィールド名を検証
    - スペースや特殊文字を除去して比較
  - 行数の上限: 1,000行
    - 超過時は分割を推奨するメッセージ
  - 空行の自動スキップ

- **複数ファイル**:
  - 最大10ファイルまで同時アップロード可能
  - 合計サイズは10MBまで
  - 複数ファイルは順次処理

### 5.2 ドラッグ＆ドロップ機能

- **画面全体オーバーレイ**:

  - ファイルがブラウザウィンドウ上にドラッグされると半透明のオーバーレイを表示
  - 「ここにファイルをドロップ」というメッセージを中央に表示
  - ドロップ可能領域を視覚的に強調

- **モーダル内ドロップゾーン**:

  - 点線の枠で囲まれたエリア
  - アイコンと説明テキスト
  - アクティブ時（ドラッグオーバー時）の視覚的フィードバック
    - 枠色の変更
    - 背景色のハイライト

- **ドラッグ操作イベント**:
  - `dragenter`: オーバーレイ表示
  - `dragover`: デフォルト動作の防止
  - `dragleave`: オーバーレイ非表示（ブラウザウィンドウから出た場合）
  - `drop`: ファイル取得と検証

### 5.3 CSVデータ処理

- **CSVパース処理**:

  - PapaParse ライブラリを使用
  - ヘッダー行の自動検出（`header: true`）
  - 空行のスキップ（`skipEmptyLines: true`）

- **ヘッダー検証**:

  - 必須フィールドの存在確認
  - フィールド名の正規化（大文字小文字を区別せず、スペースを除去）
  - 不要なBOM（Byte Order Mark）の除去

- **データ型変換**:

  - 日付文字列 → Date オブジェクト
  - 数値文字列 → Number
  - 文字列の前後スペース除去（trim）

- **トランザクション処理**:

  - Prisma.$transaction を使用したアトミック操作
  - エラー発生時のロールバック
  - 一括処理で効率化

- **エラーハンドリング**:
  - 行単位でのエラー処理（1行のエラーが全体を中断しない）
  - エラー情報の詳細記録
    - 元のデータ
    - エラーメッセージ
    - エラー発生箇所（行番号、列名）
  - 失敗データのエクスポート機能

### 5.4 進捗表示

- **プログレスバー**:

  - 進捗率の数値表示（0-100%）
  - アニメーション付きのプログレスバー
  - 処理段階の表示（検証中、アップロード中、処理中）

- **処理結果表示**:

  - 成功件数: 正常に処理されたレコード数
  - 失敗件数: エラーが発生したレコード数
  - 処理時間: 経過時間の表示

- **失敗データエクスポート**:
  - 「失敗データをダウンロード」ボタン
  - 失敗データのCSV生成
  - 元のデータにエラー情報を追加したフォーマット
  - ブラウザでのファイルダウンロード処理

## 6. エラー処理

### 6.1 バリデーションエラー

- **ファイルサイズエラー**:

  - メッセージ: 「ファイルサイズが上限（5MB）を超えています」
  - 処理: アップロード前に検証、エラー時はファイルリストに追加しない

- **ファイル形式エラー**:

  - メッセージ: 「CSVファイル形式でないファイルは処理できません」
  - 処理: MIME Type と拡張子の検証、エラー時はトースト通知

- **必須フィールド欠落エラー**:

  - メッセージ: 「必須フィールド {フィールド名} がCSVヘッダーに存在しません」
  - 処理: ヘッダー行検証時に確認、エラー時はトースト通知

- **データ型エラー**:

  - メッセージ: 「行 {行番号} の {フィールド名} の値が不正です: {値}」
  - 処理: 各行の処理時に検証、エラー時は該当行をエラーリストに追加

- **参照整合性エラー**:
  - メッセージ: 「行 {行番号} の {フィールド名} に指定された値 {値} が見つかりません」
  - 処理: 存在チェック時に検証、エラー時は該当行をエラーリストに追加

### 6.2 処理エラー

- **権限エラー**:

  - メッセージ: 「この操作を実行する権限がありません」
  - 処理: 初期権限チェック時に表示、モーダル自体の機能を制限

- **通信エラー**:

  - メッセージ: 「サーバーとの通信中にエラーが発生しました: {エラー詳細}」
  - 処理: ネットワークエラー時に表示、リトライオプション提供

- **サーバー処理エラー**:

  - メッセージ: 「サーバー側でデータ処理中にエラーが発生しました: {エラー詳細}」
  - 処理: サーバーからのエラーレスポンス時に表示

- **トランザクションエラー**:
  - メッセージ: 「データベース処理中にエラーが発生しました: {エラー詳細}」
  - 処理: トランザクション失敗時に表示、全ての変更がロールバック

### 6.3 エラー表示とリカバリー

- **トースト通知**:

  - 成功: 緑色背景、5秒後に自動消去
  - 警告: 黄色背景、8秒後に自動消去
  - エラー: 赤色背景、ユーザーが閉じるまで表示
  - 位置: 画面右上

- **エラーデータのCSVエクスポート**:

  - 形式: 元のCSVデータ + エラーカラム追加
  - 追加カラム: `error_message`, `error_type`
  - ファイル名: `登録失敗データ_{日付}.csv`
  - ダウンロード方法: Blob生成と一時リンク作成

- **リトライ機能**:
  - 同じファイルでのリトライボタン
  - 失敗データをフィルタリングした再アップロード
  - 部分的成功の場合のステータス表示

## 7. 応答性とパフォーマンス

- **UI状態管理**:

  - アップロード中:
    - アップロードボタン無効化
    - プログレスバー表示
    - キャンセルボタンの表示
  - 処理完了時:
    - 結果サマリー表示
    - 「閉じる」ボタンに変更
    - 失敗データがある場合はダウンロードオプション提供

- **大量データ処理の最適化**:

  - バッチ処理: 100行単位のバッチ処理
  - 進捗更新: 各バッチ完了時にUI更新
  - メモリ効率: 大きなデータセットを小さな単位で処理

- **バックグラウンド処理**:
  - WebWorkerの活用: CSV解析をメインスレッドから分離
  - キャンセル可能な処理: 途中でのキャンセル機能
  - 処理状態の保存: ブラウザ更新時も状態を維持

## 8. セキュリティ

- **権限チェック**:

  - サーバーサイドでの再検証: クライアント側の権限チェックだけでなくサーバー側でも検証
  - セッション検証: 有効なセッションとユーザーIDの確認
  - グループ所有権の検証: グループIDとユーザーの関連確認

- **入力データ検証**:

  - CSVデータのサニタイズ: 特殊文字や制御文字の除去
  - SQLインジェクション対策: Prismaのパラメータ化クエリ使用
  - 文字列長制限: 各フィールドの適切な長さ制限

- **アップロードファイル検証**:

  - MIME Type検証: Content-Typeヘッダーの検証
  - ファイルサイズ制限: 5MB上限の強制
  - ファイル内容検証: CSVフォーマット確認

- **クロスサイトスクリプティング対策**:
  - HTML特殊文字のエスケープ: `<`, `>`, `&`, `"`, `'` などの文字をエンティティに変換
  - Content Security Policy: インラインスクリプト実行の制限
  - HTTPヘッダーセキュリティ設定: XSS保護ヘッダーの設定

## 9. インタラクションフロー詳細

### 9.1 モーダルオープン時

1. モーダルコンポーネントがマウント
2. `useCsvUpload` フックの初期化
3. セッションからユーザー情報取得
4. 権限チェック実行
   - アプリケーションオーナー権限確認
   - グループオーナー権限確認
5. 権限に基づいてUI要素の有効/無効を設定
6. ドロップゾーンの初期化
7. グローバルイベントリスナーの設定
   - ドラッグイベント（enter/over/leave/drop）の監視

### 9.2 アップロードタイプ選択時

1. ラジオボタンクリックイベント発生
2. 選択したタイプに応じて状態を更新
3. フォーマット情報表示を更新
   - サンプルCSVデータの生成
   - フィールド説明の更新
   - 必須/オプションフィールドの表示更新

### 9.3 ファイルアップロード時

#### ドラッグ＆ドロップ

1. ファイルをドラッグするとグローバルオーバーレイ表示
2. ドロップゾーンにドラッグオーバーで視覚的ハイライト
3. ファイルドロップ時にイベントハンドラー (`onDrop`) 実行
4. ファイル検証
   - サイズチェック
   - 形式チェック
5. 検証通過したファイルをリストに追加

#### ファイル選択ボタン

1. 「ファイルを選択」ボタンクリック
2. ファイル選択ダイアログ表示
3. ファイル選択後に `onDrop` ハンドラー実行
4. ファイル検証と追加

### 9.4 アップロード実行時

1. 「アップロード」ボタンクリック
2. UI状態を「アップロード中」に変更
3. 各ファイルに対して順次処理:
   - ファイルの読み込み（FileReader API）
   - CSVパース（PapaParse使用）
   - ヘッダー検証
   - 行データの処理:
     - バリデーション
     - データ型変換
     - 参照整合性チェック
   - サーバーへのデータ送信
     - アップロードタイプに応じたServer Actionの呼び出し
     - 選択したタイプに基づく処理関数の実行
   - 結果の受信と処理
4. 進捗状況の更新
5. 完了時の結果表示
   - 成功件数/失敗件数
   - 処理時間
   - エラーがある場合のダウンロードオプション

### 9.5 エラー処理とリカバリー

1. エラー発生時の処理:
   - ファイルレベルエラー: トースト通知と処理中断
   - 行レベルエラー: エラーリストに追加して処理継続
2. 失敗データのエクスポート:
   - 「失敗データをダウンロード」ボタンクリック
   - エラー情報を含むCSVの生成
   - ファイルダウンロードの実行
3. リトライオプション:
   - エラー修正後の再アップロード
   - 部分成功の通知

### 9.6 モーダルクローズ時

1. 「キャンセル」または「閉じる」ボタンクリック
2. アップロード中の場合、確認ダイアログ表示
3. グローバルイベントリスナーのクリーンアップ
4. 状態のリセット
5. モーダルのアンマウント

## 10. 技術仕様

### 10.1 フロントエンド

- **React Dropzone**:

  - バージョン: 最新安定版
  - 設定オプション:
    - `accept: ACCEPTED_FILE_TYPES`
    - `maxSize: MAX_FILE_SIZE`
    - `multiple: true`
  - 提供メソッド:
    - `getRootProps`
    - `getInputProps`
    - `isDragActive`
    - `open`

- **PapaParse**:

  - バージョン: 最新安定版
  - パース設定:
    - `header: true`: ヘッダー行をフィールド名として使用
    - `skipEmptyLines: true`: 空行をスキップ
    - `complete`: 完了コールバック
    - `error`: エラーコールバック
  - エクスポート機能:
    - `unparse`: JSONデータをCSV文字列に変換

- **React Hooks**:

  - `useState`: 状態管理
    - `uploadType`: 選択されたアップロードタイプ
    - `isUploading`: アップロード中フラグ
    - `uploadProgress`: アップロード進捗率
    - `currentFiles`: 選択されたファイル一覧
  - `useEffect`: 副作用処理
    - 権限チェック
    - グローバルイベントリスナー設定/解除
  - `useCallback`: メモ化されたコールバック
    - ドロップハンドラー
    - アップロード関数

- **カスタムフック**:
  - `useCsvUpload`: アップロードロジックのカプセル化
    - 入力: `groupId`, `isOpen`, `onCloseAction`
    - 出力:
      - 状態オブジェクト（`uploadType`, `isUploading`, `files`等）
      - ハンドラー関数（`onUpload`, `onRemoveFile`等）
      - ドロップゾーンProps

### 10.2 バックエンド

- **Nextjs Server Actions**:

  - `bulkCreateTasks`: タスク一括作成
  - `bulkUpdateTaskStatuses`: タスクステータス一括更新
  - `bulkUpdateFixedEvaluations`: 固定貢献度一括更新
  - `bulkCreateTaskContributions`: 貢献度評価一括作成

- **Prisma ORM**:

  - トランザクション処理:
    - `prisma.$transaction`: アトミックな操作を保証
    - ネストされたデータ作成: リレーションを含む一括作成
  - エラーハンドリング:
    - 一意性制約違反の検出
    - 参照整合性エラーの処理

- **データ検証**:
  - フロントエンド: 基本バリデーション
  - バックエンド: 詳細な検証と整合性チェック
    - ユーザー存在チェック
    - タスク存在チェック
    - グループ関連性チェック
    - 権限確認

## 11. 制限事項

- **ファイルサイズ制限**:

  - 単一ファイル: 最大5MB
  - 複数ファイル合計: 最大10MB
  - 理由: サーバー負荷とメモリ使用量の制限

- **行数制限**:

  - 最大1,000行/ファイル
  - 理由: 処理時間の制約とブラウザのメモリ制限

- **エラーリカバリー**:

  - 部分的成功: 一部の行のみが成功した場合、失敗した行のみを再処理可能
  - トランザクション単位: 全て成功するか全て失敗（ロールバック）

- **ブラウザ互換性**:

  - サポート対象: 最新のChrome, Firefox, Safari, Edge
  - 非サポート: IE11およびレガシーブラウザ

- **ファイル形式**:
  - サポート: CSVのみ
  - 非サポート: XLSX, TSV, その他の表形式
  - 理由: 処理の一貫性とセキュリティリスクの軽減
